# 소방서 일일상황보고서 자동 추출·대시보드 시스템 PRD

**문서 버전**: 1.0  
**대상 문서**: 소방활동 일일 상황보고 (JPG/PNG/PDF)  
**기술 기본스택**: HTML, JavaScript, CSS 기반 SPA (모달 활용으로 페이지 이탈 없음)

---

## 1. 서비스 개요

### 1.1 목적

소방서에서 작성하는 **일일상황보고서**(이미지·PDF)를 업로드하면 **OCR·PDF 파싱**으로 내용을 자동 추출하고, **검수·수정·확정** 후 DB에 저장해 **대시보드**와 **지도**로 조회·분석할 수 있도록 하는 시스템을 구축한다.

### 1.2 주요 가치

- **수작업 입력 감소**: 보고서 필드 자동 추출으로 입력 시간 단축
- **데이터 구조화·재사용**: 표준 스키마로 저장해 통계·연계 활용
- **기간/유형별 통계·지도 시각화**: 출동 현황, 인명·재산 피해, 주요 사건 위치를 한눈에 파악

### 1.3 대상 문서

- **형식**: 「소방활동 일일 상황보고」 또는 「119 소방활동 일일 상황보고」 형식
- **파일 형식**: JPG, PNG, PDF
- **예시**: 춘천소방서 현장대응단 보고서 (예: 2026.3.3.(화) 07:00 ~ 2026.3.4.(수) 07:00, 119출동 57건 등)

## 3. 화면/메뉴 구조 (SPA 전제)

본 시스템은 **단일 페이지 애플리케이션(SPA)**으로 구성하며, 메뉴 전환 시 **전체 새로고침 없이** 메인 콘텐츠 영역만 교체한다. 상세·이력·로그 등은 **모달**로 표시하여 **페이지 이탈 없이** 동작한다.

### 3.1 메인 레이아웃

- **상단 또는 사이드 네비게이션**: 메뉴 클릭 시 `#app`(또는 동일 역할의 메인 영역)만 라우트에 따라 갱신
- **메인 콘텐츠 영역**: 목록 / 상세·검수 / 대시보드 / 지도 뷰가 동적으로 렌더

### 3.2 메뉴 구조

| 메뉴 | 경로(예시) | 설명 |
|------|------------|------|
| **보고서 목록** | `#/list` | 업로드 일시·기관·보고기간·상태(미검수/검수중/확정) 목록, **업로드** 버튼 |
| **보고서 상세/검수** | `#/report/:id` | 원본 뷰어 + 추출 결과 폼(섹션별). **저장**, **확정** 버튼. 수정 이력/추출 로그는 **모달** 또는 접이식 패널 |
| **대시보드** | `#/dashboard` | KPI 카드, 기간/유형 필터, 차트(출동 건수, 화재/구조/구급/기타 비율 등) |
| **지도** | `#/map` | 기간·유형 필터, 마커·목록 연동, 마커 클릭 시 사건 요약(모달 또는 인포윈도우), 주소 변환 실패 예외 표시 |

### 3.3 공통 UX

- **모달**: 사건 요약, 수정 이력, 추출 로그 등 상세 정보는 **모달(오버레이)**로 표시하여 SPA 내에서만 처리하고, **포커스 트랩**으로 접근성 확보.

## 4. 추출 대상 (검수·수정·확정 가능 필드)

아래 항목을 **구조화 필드**로 추출하며, **검수·수정·확정** 플로우에서 모든 필드를 편집 가능하게 설계한다.

| 구분 | 추출 항목 | 비고 |
|------|-----------|------|
| **보고서 기본정보** | 제목, 기관/부서, 연락처, 보고기간(시작일시·종료일시) | 예: 춘천소방서 현장대응단, 033) 248-9119 |
| **기상상황** | 종합예보, 오늘날씨, 지역상세(날씨, 최저/최고온도, 체감온도, 습도, 풍향/풍속) | 예: 맑음 -3°/12°, 체감 -2.9°, 습도 91% |
| **총 출동건수** | 119 출동건수(정수) | 예: 57건 |
| **화재/구조/구급/기타** | 일계·누계(출동, 오인, 인원, 사망, 부상, 피해(천원)) | 표 파싱 |
| **사망/부상/재산피해** | 일계·누계, 피해내역 상세(총액, 지역별 부동산/동산) | 예: 우두동 부동산 450만원, 동산 50만원 |
| **주요 소방활동 처리** | 사건별: 유형, 장소(주소), 시간범위, 상세처리내용 | 주소 → 카카오맵 지오코딩 |
| **금일 소방력 현황** | 기준일, 인력(주간/야간 현장·행정, 소계), 장비(펌프/탱크/구급/구조/기타), 참고(근무자 등) | |
| **특이사항/언론보도** | 항목 리스트(제목, 매체) | 예: 근속승진자 임명장 수여식 (강원일보) |

## 5. 기능 요구사항

### 5.1 업로드

- JPG/PNG/PDF 파일 선택 시 서버 또는 클라이언트에서 파싱 트리거
- **원본 파일**은 반드시 저장하고, 보고서 마스터와 연결(attachments)하여 다운로드·재검토 가능하도록 함

### 5.2 파싱·추출

- **이미지**: OCR 엔진(Tesseract.js 등)으로 텍스트 추출
- **PDF**: 텍스트 추출 후 동일 규칙 적용
- 정규식/규칙 기반으로 4절의 필드 매핑
- **추출 로그** 저장: 원문 스니펫, 매칭 필드, 오류 등 (검수 시 모달로 조회 가능)

### 5.3 검수·수정·확정

- 추출 결과를 **폼**으로 표시, 필드별 수정 가능
- **저장** 시 변경분만 **수정 이력** 테이블에 diff 형태로 저장 (필드명, 이전값, 신규값, 시각, 담당)
- **확정** 시 보고서 상태를 `confirmed`로 변경, 확정 시각 기록
- 원본·추출 로그·수정 이력은 요구사항·데이터 모델·화면 구조에 반드시 보존되도록 설계

### 5.4 원본·로그·이력

- **원본 파일**: 저장 및 다운로드 링크 제공
- **추출 로그**: 조회 시 모달 또는 접이식 패널로 표시
- **수정 이력**: 타임라인 형태로 조회(모달 권장)

### 5.5 대시보드

- 기간/기관/유형 필터
- KPI: 총 출동, 화재/구조/구급/기타 건수, 인명피해, 재산피해 등
- 차트: 기간별 추이, 유형별 비율

### 5.6 지도

- **카카오맵 API** 연동
- 주요 소방활동의 **주소 지오코딩** → 마커 표시
- 기간/유형 필터, **마커–목록 연동**(목록 클릭 → 해당 마커 포커스, 마커 클릭 → 해당 목록 하이라이트)
- 마커 클릭 시 **사건 요약**(유형, 시간, 장소, 요약) 모달 또는 인포윈도우
- **지오코딩 실패** 시: 목록/지도에 "주소 변환 실패" 표시, 재시도 또는 수동 좌표 입력(선택) 안내

## 8. 문서 분석 흐름

```mermaid
flowchart LR
  A[파일 업로드] --> B{파일 타입}
  B -->|이미지| C[OCR 엔진]
  B -->|PDF| D[PDF 텍스트 추출]
  C --> E[전처리/정규화]
  D --> E
  E --> F[필드 매핑 규칙]
  F --> G[구조화 데이터 생성]
  G --> H[DB 저장 + 추출로그]
  H --> I[검수 화면 표시]
  I --> J[수정/확정]
  J --> K[이력 저장/상태 갱신]
  K --> L[지오코딩 큐]
  L --> M[카카오맵 API]
  M --> N[좌표 저장 또는 실패 플래그]
```

- **OCR**: Tesseract.js(클라이언트) 또는 서버 OCR API
- **PDF**: pdf.js 또는 서버 라이브러리로 텍스트 추출
- **필드 매핑**: 섹션 헤더(기상상황, 일일 소방활동 등) 기준 블록 분리 후, 정규식으로 날짜/시간/숫자/주소 패턴 추출. 표는 행/열 인덱스 또는 키워드(일계, 누계, 화재, 구조 등)로 매핑

## 9. 지도 기능 상세

### 9.1 API

- **카카오맵 JavaScript API**: 지도 렌더링, 마커, 인포윈도우
- **카카오맵 주소→좌표 변환 REST API**: 지오코딩 (주요 소방활동 주소 → lat/lng)

### 9.2 마커

- `incident_reports` 중 `lat`, `lng`가 있는 항목만 지도에 표시
- 건수 많을 경우 **클러스터링** 검토 (P2)

### 9.3 필터

- **기간**: 보고기간(period_start ~ period_end) 또는 사용자 선택 기간
- **유형**: 화재/구조/구급/기타
- 필터 변경 시 마커·목록 동기 갱신

### 9.4 마커–목록 연동

- **목록 → 지도**: 목록 행 클릭 시 해당 마커로 지도 중심 이동, 해당 마커 강조
- **지도 → 목록**: 마커 클릭 시 인포윈도우 또는 **모달**에 사건 요약(유형, 시간, 장소, 요약) 표시, 목록에서 해당 행 하이라이트

### 9.5 주소 변환 실패 예외처리

- 지오코딩 실패 시 `geo_failed = true`, `lat`/`lng` null
- **목록**: 해당 행에 "주소 변환 실패" 뱃지 표시
- **지도**: 해당 건은 마커로 표시하지 않거나, 별도 레이어/아이콘으로 "미변환" 표시
- **재시도**: 주소 수정 후 "지오코딩 재시도" 버튼 제공. (선택) 수동 좌표 입력

## 13. 기술스펙 (HTML / JS / CSS 기반, SPA)

### 13.1 기본 원칙

- **HTML, JavaScript, CSS**를 기본으로 하며, 필요한 기능만 추가한다.
- **SPA(Single Page Application)**를 염두에 두고 설계한다.
- **모달**을 사용해 상세·이력·로그 등을 표시하며, **페이지를 벗어나지 않도록** 한다.

### 13.2 구조

- **단일 HTML 진입점**: `index.html`
- **메인 영역**: `div#app` 등. JS에서 라우트별 뷰를 동적 렌더
- **라우팅**: Hash (`#/list`, `#/report/:id`, `#/dashboard`, `#/map`) 또는 History API. 라우트 변경 시 **#app만 갱신**

### 13.3 기술

- **Vanilla HTML5, ES6+ JavaScript**(모듈 스크립트), **CSS3**
- 빌드 없이 동작 가능하게 유지, 필요 시 번들러(예: Vite) 도입 가능

### 13.4 필수 기능

| 기능 | 내용 |
|------|------|
| **라우팅** | Hash 또는 History API. 라우트 변경 시 #app만 갱신 |
| **모달** | 공통 모달 컴포넌트(제목, 본문, 닫기). 사건 요약, 수정 이력, 추출 로그는 **모달**로 열어 **SPA 내에서만** 처리 |
| **폼/상태** | 검수 화면은 report 객체 바인딩, 입력 시 임시 상태 유지, "저장" 시 API 호출 후 이력 기록 |
| **API 연동** | 업로드·추출 트리거, 보고서 CRUD, 목록/대시보드/지도 조회는 `fetch`로 REST 또는 로컬 JSON/IndexedDB |
| **지도** | 카카오맵 JS SDK 로드. 지오코딩은 REST 호출 후 좌표를 incident_reports에 저장. 마커/인포윈도우는 SDK API 사용 |

### 13.5 OCR/파싱

- **클라이언트**: Tesseract.js(이미지), pdf.js(PDF 텍스트 추출)
- **파서 모듈**: 추출 텍스트를 데이터 모델에 맞게 매핑하는 규칙·정규식 기반 모듈을 분리하여 유지보수성 확보
